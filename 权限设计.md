# 🎤 汪峰博客系统整体技术与设计报告

## 一、项目概述

本项目为汪峰个人博客与粉丝互动网站，融合博客、行程发布、评论互动及后台管理等功能。系统目标是构建一个具有艺术气质、流畅体验与完善权限体系的内容平台。

------------------------------------------------------------------------

## 二、技术栈选型

### 后端技术

  技术         用途         说明

------------ ------------ ---------------------

  FastAPI      后端主框架   高性能异步 REST API
  SQLAlchemy   ORM 层       管理 MySQL 数据表
  JWT          鉴权         实现角色身份认证
  Uvicorn      部署         生产环境 ASGI 服务

### 数据存储

- **MySQL**：存储文章、评论、用户、行程信息\

- **Redis**：缓存热点文章与会话状态\

  

------------------------------------------------------------------------

## 三、核心功能模块

### 1. 博客文章模块

- 支持 Markdown 文章编辑与渲染\

- 文章可添加分类、标签

- 游客可阅读，登录用户可点赞与评论\

  

### 2. 用户系统

-   支持注册、登录、头像上传\
-   角色分为：游客、用户、管理员、超级管理员\
-   用户可发布文章与行程，需经审核

### 3. 审核与管理后台

-   管理员审核用户投稿与行程\
-   超级管理员拥有删除与权限配置权\
-   后台采用 React + Tailwind UI 框架构建\
-   数据可视化：文章数、访问量、审核情况

### 4. 点赞与评论机制

-   点赞使用 Redis 防重复机制\
-   评论使用树形结构存储（`parent_id`）\

## 六、行业规范建议（参考社区标准）

### 博客内容规范

-   支持 Markdown，限制上传格式（仅图片）\
-   审核机制：人工审核 + 敏感词过滤\

### 后台管理规范

-   所有操作必须写入审计日志（`audit_log`）\
-   用户角色权限可配置\
-   所有关键操作需要二次确认（删除、封禁）

### API 规范

- 遵循 RESTful 标准：

  -   `GET /posts` 获取文章列表
  -   `POST /posts` 创建文章
  -   `PUT /posts/{id}` 更新
  -   `DELETE /posts/{id}` 删除

- 统一返回结构：

  ``` json
  {
    "status": "success",
    "data": {...},
    "message": ""
  }
  ```
